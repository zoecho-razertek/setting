## 1. 文件目的

本文件定義房東端 App「我的」頁下的設定流程，範圍包含：

1. 設定首頁與身份切換。
2. 個人房東與團隊身份的個人／團隊資訊。
3. App 語言、快速開鎖生物辨識、推播與首頁捷徑設定。
4. 個人房東與團隊身份的通用開門密碼及卡片管理。
5. 個人房東租約預設、帳務預設與催租相關設定。
6. 權限差異、資料來源、同步規則、例外處理與安全限制。

本文件不涵蓋：

- SaaS 後台的團隊建立、成員邀請、權限配置及派發新密碼實作。
- 電子鎖裝置頁內的訪客密碼、生物辨識細節管理。
- MH 商城、使用教學、公開資訊及客服內容的詳細規格。

## 2. 名詞定義

| 名詞 | 定義 |
|---|---|
| 個人房東 | 使用個人身份管理自己名下房源、租約、帳單與電子鎖的帳號。 |
| 團隊身份 | 使用公司／團隊身份管理被授權的房源與電子鎖。 |
| 團隊擁有者 | 團隊最高管理身份，預設具 `manage_team_cards` 等管理權限。 |
| 虛擬擁有者 | 當原唯一擁有者帳號完成刪除且未移轉所有權時，由系統建立、不可登入、不可操作的系統身份，用於維持團隊資料歸屬與稽核完整性。 |
| 團隊成員／授權帳號 | 由 SaaS 後台加入團隊，依角色及權限管理指定房源。 |
| 通用開門密碼 | 依身份獨立建立，套用至該身份符合條件電子鎖的開門密碼。 |
| 通用卡片 | 依身份獨立建立，不跨個人房東或不同團隊身份共用的卡片。 |
| 卡片指派 | 卡片與身份、持有人及適用電子鎖之間的授權紀錄。 |
| 租約預設設定 | 個人房東建立新租約時，預先帶入租約表單的身份級預設值。 |
| 帳務預設設定 | 個人房東建立新租約時，預先帶入帳單、收款及催租相關欄位的身份級預設值。 |
| 設定快照 | 建立租約時將當下預設值複製至租約；後續修改預設值不追溯既有租約。 |

## 3. 角色與權限矩陣

| 功能 | 個人房東 | 團隊擁有者 | 團隊成員／授權帳號 |
|---|---:|---:|---:|
| 查看個人資訊 | 可 | 可查看團隊身份資訊 | 可查看團隊身份資訊 |
| 編輯個人姓名、生日、手機 | 可；手機修改需 OTP | 不適用或由 SaaS 管理 | 不適用或由 SaaS 管理 |
| 查看團隊名稱、身份、職位、聯絡電話 | 不適用 | 可 | 可 |
| 編輯自己的團隊聯絡電話 | 不適用 | 可，且同步 SaaS 通訊錄 | 可，且同步 SaaS 通訊錄 |
| 編輯他人成員資料 | 不適用 | 依 SaaS 權限 | 依 SaaS 權限 |
| 編輯團隊名稱／圖示／公司電話 | 不適用 | 需具 `edit_team_info` | 需具 `edit_team_info` |
| 刪除帳號 | 可，受租約與團隊擁有者規則限制 | 可，受租約、訂閱與團隊接管規則限制 | 可，受租約狀態限制 |
| 查看／變更通用開門密碼 | 可查看、可變更 | 可查看、不可於 App 變更 | 可查看、不可於 App 變更 |
| 新增／刪除自己的通用卡片 | 可 | 可，預設具 `manage_team_cards` | App 端不可新增／刪除，由 SaaS 管理 |
| 指派他人成員卡片 | 不適用 | 僅 SaaS | 僅 SaaS |
| 查看卡片 | 可 | 可 | 僅顯示卡片名稱與遮罩 ID |
| App 設定 | 可 | 可 | 可 |
| 租約設定 | 可 | 無，改由 SaaS 全域設定 | 無，改由 SaaS 全域設定 |
| 切換身份 | 有多身份時可 | 有多身份時可 | 有多身份時可 |

> 權限判斷必須由後端回傳，不可只依前端身份名稱推斷；正式 permission key 由工程端依既有權限架構命名。

## 4. 導航與資訊架構

### 4.1 入口

使用者由底部導覽列進入：

`我的 → 設定首頁`

### 4.2 個人房東設定首頁

設定首頁包含：

1. 身份卡片。
2. MH 商城橫向商品區。
3. 「關於個人房東」功能群組：
   - 個人資訊。
   - 密碼與開鎖方式。
   - 租約設定。
4. 「關於 MH Lock」功能群組：
   - App 設定。
   - 使用教學。
   - 公開資訊。
5. 登出。

### 4.3 團隊設定首頁

設定首頁包含：

1. 團隊身份卡片。
2. MH 商城橫向商品區。
3. 「關於團隊」功能群組：
   - 團隊資訊。
   - 密碼與開鎖方式。
   - 團隊成員。
4. 「關於 MH Lock」功能群組：
   - App 設定。
   - 使用教學。
   - 公開資訊。
5. 登出。

團隊身份不顯示「租約設定」，其預設值由 SaaS 後台執行全域設定。

## 5. 身份切換

### 5.1 顯示條件

當帳號同時具有個人房東與一個以上團隊身份時，身份卡片可點擊，開啟「切換房東身份」選單。

選單需列出個人房東、使用者已加入的所有團隊，以及各身份名稱、身份類型及目前選取狀態。

### 5.2 切換行為

1. 使用者選取身份後，App 切換至該身份的資料範圍及功能內容。
2. App 必須記住最後一次使用的身份；下次登入或重新開啟 App 時自動進入該身份。
3. 身份切換後重新取得房源、租約、帳單、修繕、預約、承租方、電子鎖、電表、訊息與未讀數。
4. 訊息對話、未讀數、首頁捷徑與卡片均依身份隔離。
5. 卡片不得跨個人房東與不同團隊身份共用；相同實體卡若需在另一身份使用，必須於該身份重新建立卡片及授權紀錄。
6. 語言與推播分類偏好為帳號級；通知提示音、OS 通知權限及快速開鎖生物辨識為裝置級。
7. 房東端的 SaaS 訂閱與電子鎖內 IAP 訂閱為不同系統：房東端可有 SaaS 訂閱，但不提供電子鎖內 IAP 功能訂閱。
8. 商城顯示內容可依身份或後端配置更新，但購物資料不得與房源、租約等管理資料混用。

### 5.3 資料隔離

- 個人房東資料與各團隊資料必須依管理主體隔離。
- 切換身份不得搬移或複製房源、租約、帳單、訊息或卡片。
- App 不得同時混合顯示不同身份資料。
- 身份切換中的 API 請求必須取消、失效或以身份版本號防止舊資料回寫。

### 5.4 失敗處理

身份切換失敗時保留原身份及原頁面資料，顯示錯誤提示，且不得出現身份已切換但資料仍屬前一身份的狀態。

## 6. 個人資訊－個人房東

### 6.1 頁面欄位與儲存方式

| 欄位 | 顯示方式 | 操作 |
|---|---|---|
| 頭像 | 顯示目前頭像 | 可新增、替換或刪除 |
| MH 帳號 | Email 遮罩 | 唯讀 |
| 密碼 | 不顯示密碼內容 | 點擊進入密碼變更流程 |
| 姓名 | 未填時顯示「尚未填寫」 | 進入獨立編輯頁，儲存成功後立即生效 |
| 生日 | 未填時顯示「尚未填寫」 | 進入獨立編輯頁，儲存成功後立即生效 |
| 手機號碼 | 遮罩顯示 | 修改需 OTP；驗證成功後才更新 |
| Google／LINE／Apple | 顯示綁定狀態 | 可綁定／解除，受最後登入方式保護 |
| 刪除帳號 | 危險操作區 | 點擊進入刪除帳號檢查 |

手機 OTP 規則：有效 5 分鐘、60 秒後可重發、每小時最多 5 次、每日最多 10 次；手機號碼須唯一。OTP 驗證前保留原手機號碼。

生日保存完整年月日，可修改、清空且不限修改次數；不顯示年齡，未填生日則不發送生日通知。

### 6.2 社群帳號綁定與解除

1. 支援 Google、LINE、Apple。
2. 使用者必須先登入目前 MH 帳號，再從個人資訊主動綁定第三方帳號。
3. 不得只因第三方 Email 與 MH 帳號相同而自動合併帳號。
4. 第三方登入時若發現 Email 相同但尚未綁定，提示使用既有登入方式進入後再綁定。
5. 第三方唯一識別碼已綁定其他 MH Lock 帳號時，禁止重複綁定。
6. 解除綁定前必須保留至少一種登入方式；否則顯示「無法解除綁定」。
7. 解除綁定前需重新驗證：有 MH 密碼者輸入 MH 密碼；只有社群登入者重新執行任一已綁定 OAuth。生物辨識不得單獨作為解除驗證。
8. 重新驗證有效期為 5 分鐘。
9. 綁定與解除均寫入稽核紀錄；Apple 隱藏 Email 以 Apple 唯一識別碼為準。

### 6.3 頭像與團隊圖示規格

支援 JPG、JPEG、PNG、HEIC；原始檔上限 10 MB，前端上傳前將長邊壓縮至 1080 px，輸出 JPEG 或 WebP。裁切比例 1:1，建議輸出 512 × 512 px。刪除後顯示系統預設圖，失敗時保留原圖。

### 6.4 刪除帳號前置檢查

系統檢查：

1. 是否存在進行中租約。
2. 是否為任一團隊擁有者。
3. 團隊是否仍有有效 SaaS 訂閱。

「待綁定租約」不算進行中租約；除此之外，待確認押金、租客已簽回、即將搬入、租約中、屆期、提前終止、退租及其他尚未完成的租約主狀態均視為進行中。租約已結束後，即使仍有帳單或修繕資料，也不阻擋刪除。

不檢查待對帳帳單、未結修繕單、電子鎖綁定、未退款訂單或一般 IAP 訂閱。

若帳號為團隊擁有者且團隊仍有 SaaS 訂閱，禁止刪除。若無訂閱且無進行中租約，允許刪除；原唯一擁有者刪除後，由系統建立虛擬擁有者接管團隊。

虛擬擁有者規則：

- 不具登入憑證、不可操作、不可接收通知，也不得被前端當作真人顯示。
- 只維持團隊、歷史資料與稽核紀錄的合法歸屬。
- 團隊若日後恢復營運，必須由平台管理流程指定新的真人擁有者後，才可進行需擁有者權限的操作。
- 虛擬擁有者不得繞過訂閱、法務或資料刪除限制。

### 6.5 刪除帳號冷靜期

1. 送出申請後狀態設為 `pending_deletion`，進入 30 天冷靜期。
2. 所有裝置 Session 與 Token 立即失效，其他裝置同步登出。
3. 停止行銷與一般推播，但保留必要的帳號安全及法務通知。
4. 冷靜期內登入時先顯示：
   - 標題：`取消刪除帳號？`
   - 內文：`登入後將取消帳號刪除申請，並恢復帳號使用。`
   - 按鈕：`返回`／`取消刪除並登入`
5. 只有點擊「取消刪除並登入」才取消申請並恢復完整功能。
6. 冷靜期內採軟刪除標記；期滿後刪除或去識別直接識別個資，歷史租約、財務、設備安全與稽核資料依各模組保存年限保留。

## 7. 個人資訊－團隊身份

### 7.1 資料來源

團隊身份頁資料主要來自 SaaS 後台，不等同於個人帳號資料。顯示 MH 帳號、姓名、團隊名稱、帳號身份、職位與聯絡電話。

### 7.2 聯絡電話與編輯權限

1. 每位成員可修改自己的團隊聯絡電話，修改後同步 SaaS 通訊錄，不需 OTP。
2. 修改其他成員資料需 SaaS 對應權限。
3. 團隊公司電話屬團隊資訊，需具 `edit_team_info`。
4. 個人帳號手機號碼與團隊聯絡電話為不同欄位，彼此不自動同步。

### 7.3 編輯團隊資訊

具 `edit_team_info` 者可修改團隊名稱、團隊圖示與公司電話。儲存前驗證必填欄位，成功後刷新設定首頁身份卡片。

## 8. App 設定

### 8.1 設定層級

| 設定 | 層級 |
|---|---|
| 系統語言 | 帳號級 |
| 推播分類偏好 | 帳號級 |
| 首頁捷徑 | 身份級 |
| 通知提示音 | 裝置級 |
| OS 通知權限 | 裝置級 |
| 快速開鎖生物辨識 | 帳號＋裝置級，跨身份沿用但不跨裝置同步 |

原型中的「每次開啟 App，自動打開快速開鎖頁面」與「新增模式」設定均移除。

### 8.2 語言設定

本版僅支援繁體中文。未來增加語言時，儲存後立即更新目前畫面；找不到翻譯字串時以繁體中文 fallback。

### 8.3 快速開鎖生物辨識

1. 僅控制快速開鎖前驗證，不用於 App 登入。
2. 開啟前確認裝置能力及 OS 設定。
3. 設定以 `account_id + device_id` 儲存，欄位包含 `platform`、`biometric_capability`、`enabled`。
4. 生物辨識失敗或取消時不開鎖。

### 8.4 推播設定

1. 推播總開關可關閉租約、帳務、門鎖異常、緊急及其他 OS Push。
2. 關閉只停止 Push 傳送，不影響系統事件、訊息、App 內未讀與稽核紀錄。
3. 推播分類偏好為帳號級；通知提示音及 OS 權限為裝置級。
4. 關閉總開關時子項保留原值但停用；重新開啟後恢復。
5. OS 權限關閉時顯示「系統通知已關閉」，並提供「前往設定」。
6. App 回到前景時重新讀取 OS 權限。

### 8.5 首頁捷徑

可選：待辦租約、待收帳單、訂金、修繕單、預約系統、本月收入、房源狀態。

1. 捷徑偏好與排序為身份級。
2. 切換身份後，後端依權限與 SaaS 權益回傳可用項目；前端過濾不可用項目。
3. 若過濾後全部不可用，系統自動顯示該身份預設可用的第一個捷徑；不得呈現空首頁。
4. 可全部啟用，至少保留 1 個；可拖曳排序。
5. 個人房東與團隊身份的 SaaS 訂閱權益可控制預約系統；「房東端無訂閱」僅指無電子鎖內 IAP 訂閱。
6. 全部關閉時阻擋：
   - 標題：`請保留至少一個捷徑`
   - 內文：`首頁至少需要顯示一個捷徑，請選擇後再儲存。`
7. 未儲存離開提示：
   - 標題：`尚未儲存設定`
   - 內文：`離開後，本次修改將不會保留。確定要離開嗎？`
   - 按鈕：`繼續編輯`／`確定離開`

### 8.6 建立房源與租約的表單模式

App 設定頁不提供模式選項。第一次成功建立房源／租約時使用引導模式，完成後分別寫入 `has_completed_property_onboarding` 與 `has_completed_lease_onboarding`；之後固定使用快速表單，即使資料被刪除也不重置。流程中不可臨時切換。

## 9. 租約設定－個人房東

### 9.1 適用身份與入口

僅個人房東身份顯示「租約設定」，入口為 `我的 → 租約設定`。團隊端由 SaaS 後台設定。資料模型正式命名為 `LeaseDefaultSettings`。

### 9.2 共通生效規則

設定只作為新增租約的預設值，建立時複製成租約快照。新增流程可針對單筆修改，不反向更新預設；後續修改不追溯既有、待綁定、租約中或已結束租約。

### 9.3 租約設定首頁

包含租賃模式、押金、違約金、繳納指定費用、電子鎖共享許可、房源顯示、迎賓詞、帳務預設、系統模組及催租方案；移除「續約」設定。

### 9.4 押金與違約金

1. 月租可選無、1 個月、2 個月或自訂整月；僅接受正整數月數。
2. 日租不收押金，也不收違約金，因此日租模式隱藏這兩項。
3. 月租金變更時，尚未成立租約的預覽同步重算。

### 9.5 租賃模式

月租產生週期帳單；日租產生一次性帳單。日租允許綁定付款、無續約、有催租功能；上述差異由新增租約與帳單流程套用，本頁僅控制預設模式及欄位顯示。

### 9.6 繳納指定費用

1. 本版僅支援押金與首期租金。
2. 首期租金指租約成立後第一張正向租金帳單的應付淨額，不包含押金及獨立帳單。
3. 經訂金或負向帳單折抵後為 0 元，視為完成該項付款條件。
4. 日租沒有押金選項；若啟用綁定付款，只能選首期租金。

### 9.7 繳費期限

月租僅支援每月 1、5、10、15、20、25 日，預設 5 日。日租依日租帳單規格產生固定到期日。

### 9.8 收款方式

1. 不提供現金選項。
2. 支援銀行轉帳與自訂收款方式。
3. 銀行帳號非編輯頁僅顯示後五碼。
4. 已被待綁定或進行中租約引用者只能停用，不可硬刪；歷史租約使用快照。
5. 帳務模組啟用時至少保留一項啟用中的預設收款方式。
6. 資料層的刪除採軟刪除。

### 9.9 繳費通知

1. 單次提醒，範圍 0～30 天，0 代表到期日。
2. 若帳單建立時已進入提醒區間但尚未到期，建立後立即發送一次。
3. 已收、刪除或作廢帳單取消尚未發送的提醒。
4. 通知天數寫入租約快照。

### 9.10 系統模組設定

本版固定 enum 至少包含 `billing`、`repair`、`reservation`、`meter`、`messaging`。前端未知 enum 忽略並記錄，不得由後端任意增加前端無法理解的功能。

關閉帳務模組時：禁用需付款綁定、繳費期限、收款方式、繳費通知、催租語音及限用指定密碼；租約只保存租金條款，不產生 App 帳單。

### 9.11 催租語音

1. 超過繳費期限即為逾期。
2. 逾期後每次開門均由硬體裝置播放催租語音。
3. 無法針對單筆租約停用。
4. 僅在租約快照啟用且硬體支援時生效。

### 9.12 限用指定密碼

1. 指定密碼只在租客端顯示，房東不可查看。
2. 租客點擊「通知已付款」即解除限用密碼，恢復原本租客密碼，不等待房東對帳。
3. 設備離線超過 10 天，視為未繳租並啟用限用密碼。
4. 緊急授權優先於限用密碼。
5. 離線設備重新連線後套用最新狀態。

### 9.13 電子鎖共享許可

建立時形成快照。關閉時禁止新增共享；租約中變更時，依租約變更規格處理既有共享成員與生效時點，不由本頁直接撤銷。

### 9.14 房源顯示設定

即使顯示全部，API 必須回傳：

```yaml
selectable: boolean
disabled_reason_code: string | null
```

前端不得自行推算。不可選原因至少涵蓋重疊租約、無電子鎖、封存、刪除、無權限、待綁定、資料未完成及租期重疊。

### 9.15 迎賓詞

上限 100 字，允許 Emoji、常用標點與換行；純文字、不解析 HTML，URL 只當文字。每位承租方首次完成綁定後顯示一次，內容形成租約快照。

### 9.16 欄位相依與驗證

| 條件 | 系統行為 |
|---|---|
| 日租 | 隱藏押金、違約金與續約欄位；保留綁定付款及催租設定。 |
| 押金為無 | 指定費用不可勾選押金。 |
| 帳務模組關閉 | 禁用所有帳務與催租相依設定，租約不產生帳單。 |
| 無啟用中的收款方式 | 帳務模組啟用時禁止儲存。 |
| 迎賓詞空白 | 啟用狀態下禁止儲存。 |
| API 未載入完成 | 編輯入口保持 Loading／Disabled，不以空值覆蓋遠端資料。 |

### 9.17 驗收條件

1. 個人房東顯示租約設定，團隊身份不顯示。
2. 設定只影響後續新增租約。
3. 月租押金與違約金只接受整月；日租不顯示。
4. 指定費用僅有押金及首期租金。
5. 收款方式不提供現金。
6. 繳費日期只可選 1、5、10、15、20、25 日。
7. 帳務模組關閉後不產生 App 帳單。
8. 催租語音於每次開門播放；限用密碼遵守租客端顯示、付款通知解除、離線十天與緊急授權優先規則。
9. 移除續約開關及相關資料欄位。

## 10. 密碼與開鎖方式

### 10.1 功能範圍

此頁包含卡片管理及開門密碼。通用密碼與卡片均依身份獨立，不跨身份共用；訪客密碼仍在單一電子鎖下管理。本版移除掌靜脈入口。

### 10.2 套用範圍與時點

通用密碼與卡片自動套用至：空置中、未提前授權的待入住，以及公區門鎖。

- 建立待入住租約：保留房東通用憑證。
- 完成提前授權：立即建立撤銷同步任務。
- 租約起始日 00:00：撤銷房東通用憑證。
- 租約進入退租中：不恢復。
- 租約正式結束且電子鎖權限收回完成：重新加入。
- 租約結束但鎖離線：顯示「等待恢復房東權限」，不可標記為已生效。
- 未來新增符合條件的電子鎖時，自動補同步。

### 10.3 密碼顯示安全

1. 預設遮罩；按住顯示、放開隱藏，且不可選取或複製。
2. 每次首次查看完整密碼時，要求生物辨識或帳號驗證；成功後 5 分鐘內可重複查看。
3. App 進背景後立即使近期驗證失效並遮罩。
4. Android 使用 `FLAG_SECURE`；iOS 錄屏、AirPlay 與 App Switcher 顯示隱私遮罩。
5. 完整密碼不得進入剪貼簿、log、Analytics、Crash report 或狀態保存。

## 12. 個人房東開門密碼

### 12.1 變更規則

目前密碼、新密碼及確認密碼皆必填；新密碼為 6～8 位數字。後端更新成功後建立設備同步任務。

### 12.2 離線與版本控制

1. 允許延遲同步；離線鎖重新連線前舊密碼仍有效。
2. 不阻擋再次修改，但每把鎖只保留最新目標版本。
3. 24 小時未同步發送一次提醒；7 天未同步標記為高風險。
4. 明確顯示「舊密碼可能仍可使用」及尚未同步的電子鎖清單。
5. 支援手動重試同步。

### 12.3 防暴力破解

目前密碼連續錯誤 5 次後鎖定 15 分鐘，由後端跨裝置維護；每次錯誤採漸進式延遲並限制帳號與 IP 頻率。成功驗證後歸零。

## 13. 卡片管理

### 13.1 身份級與上限

卡片每個身份獨立建立，不跨身份共用。同一實體卡套用多把鎖仍算 1 張。

| 身份 | 上限 |
|---|---:|
| 個人房東 | 每個個人房東身份 15 張不同實體卡 |
| 一般團隊成員 | 每個帳號於該團隊 1 張 |
| 團隊總上限 | 每個團隊 50 張不同實體卡 |

團隊額度計入 `active`、`syncing`、`pending_delete`；全部設備完成刪除後才釋放。歷史資料不占額度。

### 13.2 卡片資料與名稱

1. 卡片名稱 1～20 字，可重複，允許中文、英文、數字、空格及常用符號；去除首尾空白且不得全空白。
2. 名稱依身份／卡片指派保存，並同步至電子鎖。
3. 同步改名失敗時保留雲端最新名稱並顯示設備同步狀態，允許重試。

### 13.3 App／SaaS 權限

- 個人房東與團隊擁有者可在 App 管理自己的卡片。
- 團隊子帳號 App 只讀；SaaS 新增／指派時檢查帳號 1 張與團隊 50 張上限。
- 硬體新增也必須通過後端權限與上限檢查。

### 13.4 NFC 新增

1. Android 點擊新增後直接進入 NFC 感應畫面。
2. iOS 每次進入前強制顯示：
   - 標題：`iOS 感應卡片`
   - 內文：`因 Apple 裝置本身的限制，iOS 不開放讀取所有類型的卡片，因此在 App 中加卡可能會失敗。若遇到無法讀取的情況，請改由電子鎖主機或門外感應裝置加卡。`
   - CTA：`仍要嘗試感應卡片 >>`
3. iOS CTA 點擊後進入 NFC 感應畫面，不導向商城。
4. 本版不提供獨立「查詢卡片」功能，所有文案移除「查詢卡片」。
5. 新增成功後自動套用至該身份的空置中、未提前授權待入住及公區門鎖；未來符合條件的新鎖自動補同步。

### 13.5 硬體新增

1. App 或 SaaS 建立一次性 `card_enrollment_session`。
2. 指定身份、持有人與目標鎖。
3. 使用者在主機／門外裝置感應卡片。
4. 設備回報卡片 ID，後端檢查重複、權限與上限。
5. 成功後建立身份級卡片並同步其他適用鎖。
6. 硬體新增完成後 App 可下拉更新；手機端新增成功則自動刷新。

### 13.6 刪除與同步狀態

狀態：

- `deleted`：所有電子鎖同步完成。
- `pending_delete`：部分鎖離線，卡片可能仍可使用。
- `delete_failed`：在線鎖同步失敗，可重試。

提示：

- 標題：`部分電子鎖尚未完成刪除`
- 內文：`卡片已從帳號移除，但仍有電子鎖尚未連線。同步完成前，此卡片可能仍可開啟部分門鎖。`
- 按鈕：`我知道了`／`查看設備`

本版無掛失與重新啟用；遺失一律刪除。

10. 正式 permission key 由工程端命名，但至少涵蓋團隊資訊、通用密碼、卡片查看／管理與派發新密碼。

1. 帳號、手機、卡片 ID 與開門密碼遮罩顯示。

## 19. 驗收條件

### 19.1 身份與設定層級

1. 身份切換後資料、訊息、未讀、捷徑及卡片均正確隔離。
2. 語言與推播偏好跨身份沿用；生物辨識及提示音不跨裝置同步。
3. 個人房東與 SaaS 訂閱、電子鎖 IAP 權益不混用。

### 19.2 帳號與個人資訊

1. 姓名、生日可即時儲存；手機需通過 OTP。
2. 社群帳號不依 Email 自動合併，解除最後登入方式會被阻擋。
3. 刪除帳號會檢查進行中租約與團隊 SaaS 訂閱；待綁定租約不算進行中。
4. 冷靜期登入需明確確認取消刪除。
5. 唯一擁有者刪除後建立不可登入的虛擬擁有者。

### 19.3 App 設定

1. 首頁捷徑依身份與權益過濾且至少保留一項。
2. 首次成功建立後寫入 onboarding flag，後續使用快速表單。
3. 推播關閉不影響 App 內事件與未讀。

### 19.4 通用密碼與卡片

1. 通用憑證依身份獨立，按房源狀態於正確時點新增／撤銷。
2. 離線密碼 24 小時提醒、7 天高風險；再次變更只保留最新版本。
3. iOS 說明 CTA 進入 NFC 感應畫面，且文案不含「查詢卡片」。
4. 個人房東 15 張、團隊成員帳號級 1 張、團隊 50 張；`pending_delete` 仍占額度。
5. 刪除卡片時清楚提示離線鎖可能仍可使用。
6. 卡片名稱同步至電子鎖。

### 19.5 租約設定

1. 正式名稱統一為「租約設定」。
2. 日租無押金、違約金、續約；允許綁定付款及催租。
3. 本版指定費用僅押金與首期租金。
4. 不提供現金；繳費日只有 1、5、10、15、20、25 日。
5. 帳務模組關閉後不產生帳單。
6. 限用密碼只在租客端顯示，通知已付款即解除；離線超過 10 天啟用；緊急授權優先。
7. 已移除續約開關。
